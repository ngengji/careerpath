ภาพรวม: เราทำอะไร?

เริ่มจาก dashboard หน้าเดียวที่ ดูข้อมูลได้อย่างเดียว → จบที่ web app ที่ เก็บข้อมูลได้จริง ซิงค์ทุกอุปกรณ์ ใช้งานได้ทั่วโลก

Browser (React) ↕ supabase-js Supabase (PostgreSQL) ↑ GitHub Actions (build + deploy) ↑ GitHub (source code)

ขั้นตอนที่ 1 — สร้าง Frontend (หน้าตาเว็บ)

ReactTypeScriptVite

  • React — framework สร้าง UI เป็น component เล็กๆ ประกอบกัน
  • TypeScript — JavaScript ที่มี type ช่วยจับ bug ก่อน run
  • Vite — ตัว build และ dev server ให้ทำงานเร็ว

สิ่งที่ทำในขั้นนี้

  • วาดกราฟ scatter plot ด้วย SVG
  • ทำ filter, tooltip, action panel
  • ใช้ seeded random สร้างข้อมูลจำลอง 190 คน ให้ผลเหมือนกันทุกครั้ง
  • แยก data.ts เป็น single source of truth ไม่ให้ข้อมูลสองหน้าต่างกัน

ขั้นตอนที่ 2 — เพิ่ม Database (เก็บข้อมูลจริง)

SupabasePostgreSQL

ทำไมถึงเลือก Supabase?

  • มี REST API สำเร็จรูป ไม่ต้องเขียน backend เอง
  • anon key ใส่ใน frontend ได้ปลอดภัย เพราะมี Row Level Security คุม
  • Free tier เพียงพอสำหรับงาน internal

SQL ที่รันใน Supabase

-- สร้างตารางเก็บ action plan
CREATE TABLE actions (
  emp_id integer PRIMARY KEY,
  tags   text[]   DEFAULT '{}'
);

-- เปิด security แต่ให้ anon อ่าน/เขียนได้
ALTER TABLE actions ENABLE ROW LEVEL SECURITY;
CREATE POLICY "allow_anon_all" ON actions
  FOR ALL TO anon
  USING (true) WITH CHECK (true);

อย่าลืมเปิด Settings → API → Enable Data API ด้วย มิฉะนั้น REST API จะ 404

ขั้นตอนที่ 3 — เชื่อม Frontend กับ Database

@supabase/supabase-js

// อ่านข้อมูลจาก database
const { data } = await supabase
  .from("actions")
  .select("emp_id, tags");

// บันทึกข้อมูล (upsert = insert ถ้าไม่มี, update ถ้ามีแล้ว)
await supabase.from("actions").upsert({
  emp_id: 133,
  tags: ["promote", "succession"]
});

Design Pattern สำคัญ — Fallback

ลำดับการใช้งาน

ลอง Supabase ก่อน → ถ้าไม่ได้ → ใช้ localStorage แทน

ทำให้เว็บยังใช้งานได้แม้ไม่มี internet หรือ database มีปัญหา

ขั้นตอนที่ 4 — Deploy ขึ้น Internet

GitHubGitHub PagesGitHub Actions

code ในเครื่อง → git push → GitHub Actions build → GitHub Pages deploy

ไฟล์สำคัญ

ไฟล์หน้าที่
.github/workflows/deploy.ymlสั่ง GitHub ทำอะไรเมื่อมี push
.env.localAPI keys (ไม่ขึ้น GitHub!)
src/data.tsข้อมูลกลาง + functions เชื่อม database

เรื่อง Security ของ API Key

  • .env.local → ใส่ใน .gitignore → ไม่ขึ้น GitHub
  • GitHub Secrets → เก็บ key ให้ GitHub Actions ใช้ตอน build
  • anon key → ปลอดภัยเพราะ RLS จำกัดสิ่งที่ทำได้แม้ถูก copy ไปใช้

🏆 กฎทอง Vibe Coding

  • ถามทีละขั้น ไม่ต้องรู้ทุกอย่างก่อนเริ่ม
  • Error message คือ clue — copy ไปถามได้เลย
  • ทดสอบบ่อยๆ อย่ารอทำเสร็จทั้งหมดแล้วค่อยทดสอบ
  • Commit บ่อยๆ เผื่อต้องย้อนกลับ
  • ไม่ต้องเข้าใจทุก line — เข้าใจแค่ว่า "ทำอะไร" ก็พอ
